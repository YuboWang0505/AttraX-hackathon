# AttraX 换肤指导文档

> **一句话说明**:本次改动**只换视觉层(颜色 + UI 质感)**,**布局和交互一律不动**。
> 任何开发者或 AI Agent 拿到这份文档,都应该能在不改动结构的前提下完成换肤。

---

## 🎯 核心原则(读完再动手)

### ✅ 可以改的

- 颜色(背景、前景、主色、文字色、描边色)
- 按钮/卡片的**视觉表现**(渐变、阴影、模糊、质感)
- 字体字族、字重
- 图标的**视觉风格**(线性 ↔ 面性、粗细、圆润度)
- 圆角数值(但要**全局同步**改,不能只改一处)

### ❌ 绝对不能改的

- 页面数量、页面顺序
- 每个元素在屏幕上的**位置和尺寸**
- 元素的**层级关系**(谁在上、谁在下、谁弹出)
- 元素的**对齐方式**(居中/左对齐/右对齐)
- 元素间的**间距比例**
- 所有**文案内容**(一个字都不改)
- 所有**交互逻辑**(点击跳转、弹层触发、手势)
- 图标的**语义**(铅笔还是铅笔,蓝牙还是蓝牙,只能换画风)

## 📐 参考原版的结构(骨架不动)

本项目有 5 个页面,每个页面的结构必须照搬原版:

### Page 1 · 启动页
- **布局**:顶部品牌名(左对齐)→ 中部光晕 + 重复水印 → 底部 CTA 按钮(居中)
- **元素清单**(缺一不可):`Hi!` 问候语、品牌名、径向光晕、重复水印文字、START 按钮、右箭头图标
- **交互**:点击 START 按钮进入下一页

### Page 2 · 蓝牙配对页
- **布局**:左上返回 → 顶部大标题 + 副标(居中) → 中部主 CTA(居中) → 底部设备列表
- **元素清单**:返回箭头、大写主标题、中文副标、橙色推进按钮、5 行设备列表容器
- **交互**:点击列表项选设备,点击推进按钮下一步

### Page 3 · 验证码页
- **布局**:左上返回 → 大写主标题(居中) → 原始码提示行(带下划线数字 + 刷新图标) → 4 位数字药丸(横排居中) → S/M 切换器(居中) → 推进按钮(居中)
- **元素清单**:返回箭头、`YOUR CODE` 标题、4 位原始码 + 刷新 icon、4 个数字容器、S/M segmented 切换、橙色推进按钮
- **交互**:4 位数字可输入/修改,S/M 可切换,点击推进下一步

### Page 4 · 安全词设置弹层
- **布局**:全屏底部虚化聊天气泡 → 中央偏下白色模态卡 → 卡内:标题 + 灰色输入药丸(带铅笔 icon)+ 底部说明小字
- **元素清单**:背景模糊层、模态卡片、"安全词设置"标题、铅笔图标、"安全词"输入框、说明文字"美好的体验以彼此自愿、相互尊重为前提"
- **交互**:点击输入框可编辑,点击外部或确认关闭弹层
- ⚠️ **说明文字是产品伦理底色,一字不动**

### Page 5 · 聊天主界面
- **布局**:左上 X 关闭 + 右上安全词药丸 → 中部对话气泡区(左右交替) → 底部黑色消息药丸 + 右侧橙色通话圆形 FAB → 最底部弧形刻度盘露出
- **元素清单**:X 按钮、安全词药丸(带铅笔 icon)、聊天气泡(带尖角)、消息输入药丸、通话 FAB、底部弧形刻度盘(带红点和刻度线)
- **交互**:点击消息药丸打开输入,点击通话 FAB 发起通话,底部刻度盘可上滑调节
- ⚠️ **弧形刻度盘是核心产品隐喻,形态和交互必须保留**

## 🔁 执行步骤(开发/AI Agent 按顺序做)

1. **全局搜索旧色值**,确保没有硬编码的颜色残留,所有颜色都走 Token
2. **填写上方 Token 表的右侧**(由设计师/产品/用户提供新方向)
3. **替换 Token 值**,**Token 名保持不变**,代码里引用 Token 的地方一律不动
4. **逐页截图对比**,原版 vs 新版并排,检查结构是否 100% 一致
5. **交互回归测试**:点一遍所有按钮、弹层、跳转,确保流程完全不变

## ⚠️ 常见陷阱

1. **不要顺手改文案** —— "Add to Home Screen" 是原版占位,是否替换走产品决策,不属于换肤
2. **不要顺手优化间距** —— 即使觉得某处间距不合理,本次不动
3. **不要改图标语义** —— 铅笔、蓝牙、电话、箭头,只换画风,不换含义
4. **不要合并深浅模式** —— 原版深色页和浅色页是有意设计,两套氛围都保留
5. **不要动弧形刻度盘** —— 产品核心隐喻,只能换色
6. **改药丸圆角要全局同步** —— 如果要把 `999px` 改成 `12px`,所有按钮/输入框/数字位同步改,不能只改一个
